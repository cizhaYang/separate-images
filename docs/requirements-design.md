# 图片分割网页工具需求设计文档

**项目目标**

做一个纯前端网页工具。用户上传一张照片后，可以在页面中预览图片、拖动分割线调整分割位置，并按当前分割结果导出左右或上下两张图片。

## 1. 背景与目标

用户希望有一个轻量的网页工具，用来把单张照片从中间或自定义位置切成两张。工具需要满足三个核心体验：

1. 上传后能立即看到原图预览。
2. 能直接在当前页面拖动分割线，实时看到分割位置。
3. 点击确认后，把分割后的两张图片保存到本地，并尽量允许用户选择保存位置。

本项目只做前端网页版本，不做后端服务，不做账号系统，不做批量处理。

## 2. 范围

### 2.1 需要实现

- 上传单张图片。
- 在页面内预览图片。
- 支持两种分割方向：
  - 竖直分割，输出左图和右图。
  - 水平分割，输出上图和下图。
- 分割线默认位于图片中心。
- 用户可以拖动分割线调整位置。
- 预览中始终显示分割线。
- 点击确认后导出两张分割图片。
- 导出文件名按原文件名加后缀生成。
- 保存时优先尝试让用户选择保存目录；不支持时回退到浏览器下载。

### 2.2 不做的内容

- 不做图片旋转、裁剪、缩放、滤镜。
- 不做多图批量上传。
- 不做云端保存。
- 不做登录、分享、协作。
- 不做复杂编辑器，只保留分割这一项核心能力。

## 3. 用户流程

1. 用户打开页面。
2. 用户上传一张图片。
3. 系统显示图片预览，并在中心位置显示分割线。
4. 用户切换分割方向为竖直或水平。
5. 用户拖动分割线到目标位置。
6. 用户确认当前分割结果。
7. 系统生成两张图片。
8. 系统尝试让用户选择保存目录并写入文件。
9. 如果浏览器不支持目录写入，则触发两张图片的下载。

## 4. 交互设计

### 4.1 上传区

- 支持点击选择文件。
- 支持拖拽文件到上传区域。
- 只允许选择单张图片。
- 上传后立即进入编辑状态。

### 4.2 预览区

- 显示原图缩放预览。
- 预览图上覆盖一条分割线。
- 分割线位置和方向与当前设置同步。
- 拖动时即时更新预览。

### 4.3 分割方向

- 提供竖直 / 水平切换。
- 切换后分割线方向立即变化。
- 切换方向时，分割线位置保持在当前比例位置，而不是强制回到中心。

### 4.4 分割线拖动

- 竖直分割时，拖动线在图片宽度范围内移动。
- 水平分割时，拖动线在图片高度范围内移动。
- 分割线不能拖出图片边界。
- 默认位置为 50%。
- 分割位置以原图坐标比例保存，不以屏幕像素保存，避免缩放误差。

### 4.5 保存按钮

- 用户确认后触发导出。
- 导出时生成两张图片。
- 导出完成后给出简单结果提示。

## 5. 导出规则

### 5.1 导出结果

固定导出两张图片：

- 竖直分割：
  - `原文件名-left.原扩展名`
  - `原文件名-right.原扩展名`
- 水平分割：
  - `原文件名-top.原扩展名`
  - `原文件名-bottom.原扩展名`

### 5.2 格式策略

- 优先保持原图格式。
- 如果原格式不适合浏览器导出或写入，则统一降级为 PNG。
- 对照片场景，常见的 JPG/JPEG 保持原格式即可。
- 如果发生格式降级，文件扩展名同步改为 `.png`。

### 5.3 保存位置策略

- 优先使用浏览器的目录选择能力进行保存。
- 如果浏览器不支持该能力，则回退为普通下载。
- 回退下载时由浏览器决定默认保存位置，前端不强行控制系统目录。

## 6. 技术方案

### 6.1 前端栈

- Vite
- React
- 原生 Canvas

### 6.2 图像处理方式

- 图片解码后加载到 `Image` 对象或 `createImageBitmap`。
- 用 Canvas 根据当前分割比例裁出两块区域。
- 分别导出为 Blob 或文件对象。

### 6.3 保存方式

- 优先尝试 File System Access API。
- 不可用时使用 `<a download>` 触发下载。
- File System Access API 主要在 Chromium 系浏览器中可用，并要求页面运行在安全上下文中。本地开发应通过 `http://localhost` 访问，而不是直接打开 HTML 文件。

## 7. 页面结构建议

页面保持单页工具风格，结构分为三块：

1. 顶部操作区：上传、方向切换、导出按钮。
2. 中部预览区：显示图片和分割线。
3. 底部状态区：显示当前方向、分割比例、导出状态和错误提示。

## 8. 状态设计

建议维护以下状态：

- `sourceFile`：当前上传文件。
- `sourceUrl`：用于预览的本地地址。
- `imageWidth` / `imageHeight`：原图尺寸。
- `splitDirection`：`vertical` 或 `horizontal`。
- `splitRatio`：0 到 1 的分割比例。
- `isDragging`：是否正在拖动分割线。
- `exportStatus`：空闲、处理中、成功、失败。
- `exportError`：错误信息。

## 9. 边界情况

- 未上传图片时禁止导出。
- 图片加载失败时提示重新上传。
- 超小图片时分割线仍需可见，但拖动范围必须正常限制。
- 用户切换方向后，分割比例沿用当前值。
- 导出失败时保留当前编辑状态，便于重试。

## 10. 验收标准

满足以下条件即视为完成：

- 能上传一张图片并正常预览。
- 预览中能看到分割线。
- 分割线可拖动，且位置实时生效。
- 支持竖直与水平两种分割方向。
- 点击确认后能导出两张图片。
- 导出文件名符合约定。
- 浏览器支持时能选择保存目录，不支持时能正常下载。

## 11. 后续实现建议

实现时建议把能力拆成三层：

1. 交互层：上传、拖动、方向切换、状态提示。
2. 计算层：分割比例、原图坐标映射、导出文件名。
3. 导出层：Canvas 裁切、Blob 生成、保存回退策略。
